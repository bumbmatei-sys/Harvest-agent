## Fix 1: Canvas — Proper Excalidraw Whiteboard

### Design Spec (inspired by Miro + Excalidraw)

**Full-screen canvas:**
- Hide Admin header when canvas is active
- Hide bottom nav when canvas is active
- Only show: minimal floating toolbar (Back button, Canvas name, Save status, Users count)
- Excalidraw takes 100vh x 100vw

**Floating toolbar (top-left):**
- Back arrow → returns to canvas list
- Canvas name (editable inline)
- Save status: "Saving..." / "Saved ✓" / "Save failed ✗"
- Online users count badge (if others are editing)

**Excalidraw features:**
- Freehand drawing (pen tool)
- Sticky notes (rectangle with text)
- Shapes (rectangle, ellipse, arrow, line)
- Text tool
- Selection tool
- Color picker
- Undo/Redo
- Zoom in/out/fit

**Colors (Harvest design):**
- Canvas background: white
- Toolbar bg: white with shadow
- Active tool: gold `#C9963A`
- Back button: navy `#0b1121`

### Implementation

```tsx
// CanvasEditor.tsx — full rewrite
import dynamic from 'next/dynamic';
const Excalidraw = dynamic(
  () => import('@excalidraw/excalidraw').then(mod => ({ default: mod.Excalidraw })),
  { ssr: false }
);

// Full-screen layout — NO header, NO bottom nav
return (
  <div style={{ position: 'fixed', inset: 0, zIndex: 9999, background: 'white' }}>
    {/* Floating toolbar */}
    <div style={{ position: 'absolute', top: 16, left: 16, zIndex: 10000, ... }}>
      <button onClick={onBack}>← Back</button>
      <span>{canvasName}</span>
      <span>{saveStatus}</span>
    </div>
    
    {/* Excalidraw canvas */}
    <Excalidraw
      initialData={{ elements: canvasData.elements || [] }}
      onChange={handleChange}
      UIOptions={{ canvasActions: { changeViewBackgroundColor: false } }}
    />
  </div>
);
```

**Key:** The parent `AdminDashboard.tsx` should hide the nav when `canvasId` is set.

## Fix 2: Newsletter — Premium Design

### Design Spec (inspired by Notion + Resend)

**Two-column layout (desktop) / stacked (mobile):**

**Left side (60%): Editor**
- Subject line input (large, clean)
- "Generate from Instagram" button (prominent, gold)
- Content editor (TipTap rich text, not textarea)
- Or: HTML editor with syntax highlighting

**Right side (40%): Live Preview**
- Renders the newsletter HTML in a phone/email frame
- Updates in real-time as user edits
- Shows how it will look in email clients

**Top bar:**
- "New Newsletter" title
- Preview | Save Draft | Send buttons
- Connection status badges (Instagram ✓ / Mailchimp ✓)

**Generate flow:**
1. Admin clicks "Generate from Instagram"
2. Loading state: "Fetching your Instagram posts..."
3. AI generates newsletter
4. Content populates in editor
5. Preview updates on right side
6. Admin edits, then clicks Send

### Components

**ConnectionStatus** — Shows Instagram + Mailchimp connection status with connect/disconnect buttons
**SubjectInput** — Large, clean input with character count
**ContentEditor** — TipTap or simple textarea with line numbers
**LivePreview** — Iframe or sandboxed div rendering the HTML
